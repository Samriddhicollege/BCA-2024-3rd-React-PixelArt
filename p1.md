# High-Resolution Grid Performance Optimization

## The Problem
When the maximum allowed grid resolution was increased from `64x64` to `256x256` for the Pixel Art application, extreme latency and delay were observed while drawing or performing flood-fill operations. 

In a traditional React pattern, painting operations fired state updates (`setGrid`) for every single pixel hovered over. In a full `256x256` grid, that means React was being tasked with cloning, diffing, and attempting to re-render an array of **65,536 elements** sequentially multiple times every second (dozens of times per stroke). This heavy virtual DOM reconciliation process inevitably blocked the browser's main thread and crippled the frame rate.

## The Solution
To achieve a smooth 60 FPS drawing experience while maintaining React's structural integrity, we implemented **Direct DOM Mutations** combined with **Deferment and Memoization**. Features of the solution include:

### 1. Direct DOM Manipulation (Bypassing React State)
Instead of updating the React UI state for every active pixel during a cursor stroke, we sidestep component rendering completely. We maintain a mutable, non-triggering memory representation (using `useRef`) of the grid array. When a pixel is interacted with, we instantly mutate the browser's DOM element directly:
```javascript
currentGridRef.current[y][x] = newColor;
document.getElementById(`cell-${y}-${x}`).style.backgroundColor = newColor;
```

### 2. Deferred History and State Syncing
By waiting until the exact moment the stroke finishes (via the `onMouseUp` event), we can batch the entire sequence. React is securely handed the finalized version of the 2D array, ensuring our `Undo/Redo` stacks and Gallery features stay safely in sync with the DOM without ever interrupting the actual drawing process.

### 3. Component Memoization
We abstracted out individual pixels into a heavily wrapped `React.memo` cell component. This ensures that even when React attempts to sync the state post-stroke, it immediately ignores the 65,535 pixels that haven't changed—saving immense computation power. Stable callback references (`useCallback`) are heavily favored to enforce identical prop references across the grid.
